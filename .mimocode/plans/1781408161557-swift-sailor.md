# Criar Skill: Streamlit DOM Cleanup Guide

## Objetivo

Criar um arquivo `docs/streamlit-dom-cleanup.md` com os padrões e regras documentados para limpar divs desnecessárias em dashboards Streamlit. Referência reutilizável para outros projetos (ex: Hermes).

## Arquivo a criar

`docs/streamlit-dom-cleanup.md`

## Conteúdo

Documento Markdown com as seguintes seções:

### 1. `st.markdown(unsafe_allow_html=True)` vs `st.html()`
- Tabela comparativa de DOM gerado, divs extras, CSS classes, Material Symbols
- Exemplo de conversão com código antes/depois
- **Regra:** Se só usa estilos inline → `st.html()`

### 2. `st.columns(1)[0]` — Wrapper Inútil
- Por que cria divs extras (stHorizontalBlock > stColumn > stVerticalBlock)
- Como substituir com CSS via `:has()` no container pai
- Exemplo de CSS

### 3. Hidden Markers — Padrão CSS `:has()`
- Como usar divs ocultos como âncoras CSS
- Limitações do `:has()` (browser support, não funciona entre irmãos)
- Evite markers duplicados

### 4. Streamlit DOM — O que cada widget cria
- Tabela: st.columns, st.container, st.markdown, st.html, st.selectbox, st.plotly_chart, st.expander

### 5. CSS Scoping — Como Estilizar Sem Poluir
- Padrão 1: Marker + `:has()`
- Padrão 2: Posição do child (evitar)
- Padrão 3: data-st-key
- Padrão 4: data-testid (mais estável que classes Emotion)

### 6. Popover/Dropdown — Portal Fora do DOM
- CSS para `[data-baseweb="popover"]`
- Cuidado com regras globais

### 7. Material Symbols em `st.html()`
- Como carregar fonte via `<link>` no iframe

### 8. st.selectbox — Fonte e Tamanho
- CSS para trigger (`.st-bz`) e dropdown (portal)

### 9. Checklist de Limpeza
- Lista de verificações antes de considerar uma seção "limpa"
